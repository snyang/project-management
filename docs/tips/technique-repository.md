# 部门管理 - 技术知识仓库

## 建立技术知识仓库的原则

- 技术知识仓库极其容易失败，主要原因是：有人写，没人用。
- 赢得过互联网搜索引擎
  判断是否将一个知识点放到知识仓库中的原则：使用者认为使用技术知识仓库优于互联网搜索引擎。
- 使用另一个知识库B，让员工自由的编辑
- 将技术知识仓库当做一个产品来开发，确定用户，找到需求，筛选需求，按需开发
- 定期将知识库B的有用内容放到技术知识仓库中
- 系统化的知识比较适合放到技术知识仓库中

## 技术库的内容

- 建立合理的知识分类，和导航
- 学习某个技术的线路图
- 项目管理规范和流程
- 技术规范
- 文档模板
- 常用的代码
- 常用的类封装
- 架构模板
- 项目模板
